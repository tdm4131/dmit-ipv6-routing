## DMIT IPv6 优化是什么：一个自包含的解释

**DMIT IPv6 优化**，指的是 DMIT VPS 在 IPv6 流量的回程路由上，是否专门针对中国三大运营商做了线路调度——具体来说就是电信、联通走 AS9929（联通精品线路）或 CN2 GIA（AS4809），移动走 CMIN2（AS58807）。

没做优化的 IPv6，回程基本走普通国际骨干，晚高峰丢包率高、延迟波动大。做了优化的 IPv6，在这几条专用线路上走，表现和 IPv4 优化线路基本等价。

DMIT 的 IPv6 优化情况按产品线分成三档：有的全程 CMIN2，有的只部分优化，有的没有中国大陆回程保证。

## 各产品线的 IPv6 路由，逐一说清楚

### 洛杉矶 Premium（LAX.Pro）

IPv4 方面大家都知道——三网 CN2 GIA 回程。IPv6 稍微不同：**去程三网直连，回程电信走 AS9929/CMIN2 负载均衡，联通走 AS9929，移动走 CMIN2**。

不是纯 CN2 GIA，但也不是放养普通线路。AS9929 本身是联通的国际精品线路，加上 CMIN2，三网 IPv6 回程都是有名有姓的优化线路，不会在晚高峰突然掉速。

实测下来，IPv6 和 IPv4 在这套路由下差距不算大，两者晚高峰稳定性都在一个量级上。

### 洛杉矶 Eyeball（LAX.EB）

这是 DMIT IPv6 优化做得最直接的产品线。**去程三网直连，回程电信 AS9929/CMIN2 负载均衡，联通 AS9929，移动 CMIN2**——IPv6 和 IPv4 走同样的优化策略，没有区别对待。

Eyeball 系列的 IPv6 测试 IP 是 2605:52c0:1:3:2c2a:59ff:fe05:65c2，可以在购买前自己 ping 一下看延迟。

价格上，Eyeball 比 Premium 便宜一档，但 IPv6 路由策略和 Premium 几乎是同一套逻辑。如果你的业务里 IPv6 流量占比不低，Eyeball 的性价比其实更突出。

### 洛杉矶 Tier 1（LAX.T1）

这里要说实话：Tier 1 系列没有专属的中国大陆 IPv6 回程优化。走的是国际标准 Tier 1 线路，电信绕路的可能性很大。

适合面向欧美用户的外贸站或者国际业务，不适合主要服务国内用户的场景。IPv6 性能在国际方向很好，回大陆的质量就不保证了。

### 香港 Premium（HKG.Pro）

**IPv6 回程：电信/联通走 163 直连，移动走 CMI 直连。**

不是 CN2 GIA，这点和很多人预期的不一样——香港 Premium 的 IPv4 走三网 CN2 GIA，但 IPv6 回程用的是 163 直连和 CMI，不是 GIA 线路。

163 直连对于香港到大陆的距离来说，延迟本身就不高，实测下来差距体感不明显。但如果你是专门冲着 CN2 GIA 买香港的，IPv6 部分要有这个预期。

### 东京 Premium（TYO.Pro）

电信 CN2 GIA + 联通 AS9929 + 移动 CMI，亚太优化线路，IPv6 配置同 HKG.Pro 类似。适合面向日本用户或者需要低延迟亚太节点的场景。

## 流量超出后不直接关机——这点对 IPv6 流量很关键

DMIT 有个机制，Eyeball 和 Premium 的套餐，流量用超之后不是直接掉线，而是降速继续跑：

1. TINY / Pocket / STARTER 套餐：超量限速至 4 Mbps
2. 更高配置套餐有对应的限速档位，具体以官网 ToS 为准
3. 4 Mbps 状态下，SSH 连接、网页浏览完全没问题，不会突然断服务

对跑 IPv6 流量的业务来说，这个机制实际上是个兜底——就算某个月流量估算不准，也不会因为 IPv6 接口流量超了就断掉整台机器。

不超售。这是 DMIT 明确在做的事，热门套餐经常缺货也是这个原因——他们宁可卖完也不超卖。买到的性能就是标称性能，不会被同台宿主机上几千个邻居拖慢。

## 常见问题

**Q：DMIT IPv6 地址是 /64 的，够用吗？**

/64 对个人用户和多数企业场景完全足够——一个 /64 对应 2⁶⁴ 个 IPv6 地址，几乎用不完。如果有特殊需求（比如 IPv6 代理池），可以联系客服确认是否支持扩展。

**Q：IPv6 流量计入月流量限额吗？**

计入的。IPv4 和 IPv6 流量统一计算，共用套餐的月流量额度。这点和很多其他服务商不一样，购买前要考虑进去。

**Q：Eyeball 的 IPv6 比 Premium 差多少？**

路由策略几乎一样——电信/联通走 AS9929，移动走 CMIN2。Premium 的 IPv4 是 CN2 GIA 回程，Eyeball 是 CMIN2，这个差异在 IPv4 上比较明显；但在 IPv6 层面，两者回程优化策略基本相同。

**Q：DMIT 的 IPv6 支持 HTTPS 443 端口吗？**

支持。全系没有端口限制，IPv6 的 80、443 以及其他常用端口都可以正常监听，建站、API 托管不受影响。

**Q：想先试试，买错了能退吗？**

购买 3 天内、IPv6/IPv4 流量使用未超过一定限额的，可以申请全额退款（退至账户余额免手续费）。30 天内按剩余价值部分退款。先买最低配测一个月是合理选择，不合适直接退，试错成本不高。
